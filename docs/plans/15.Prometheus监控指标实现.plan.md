# Prometheus 监控指标实现

## 目标
为 ezlb 添加 Prometheus 监控指标支持，复用 admin 端口暴露 `/metrics` 端点，指标类型仅使用 Counter 和 Gauge。

## 设计要点

### 1. 指标列表

| 指标名 | 类型 | 标签 | 说明 |
|--------|------|------|------|
| `ezlb_service_connections_total` | Counter | service, listen, protocol | 累计连接数 |
| `ezlb_service_bytes_in_total` | Counter | service, listen, protocol | 入向字节数 |
| `ezlb_service_bytes_out_total` | Counter | service, listen, protocol | 出向字节数 |
| `ezlb_service_packets_in_total` | Counter | service, listen, protocol | 入向包数 |
| `ezlb_service_packets_out_total` | Counter | service, listen, protocol | 出向包数 |
| `ezlb_backend_connections_total` | Counter | service, backend, protocol | 后端累计连接数 |
| `ezlb_backend_bytes_in_total` | Counter | service, backend, protocol | 后端入向字节数 |
| `ezlb_backend_bytes_out_total` | Counter | service, backend, protocol | 后端出向字节数 |
| `ezlb_backend_active_connections` | Gauge | service, backend, protocol | 后端活跃连接数 |
| `ezlb_backend_inactive_connections` | Gauge | service, backend, protocol | 后端非活跃连接数 |
| `ezlb_backend_health_status` | Gauge | service, backend | 1=healthy, 0=unhealthy |
| `ezlb_config_reload_total` | Counter | - | 配置重载次数 |
| `ezlb_reconcile_errors_total` | Counter | - | Reconcile 错误次数 |

### 2. 配置设计

```yaml
global:
  admin_address: "127.0.0.1:9095"  # Admin 端口地址，为空则禁用
  metrics_enabled: true            # 是否启用 metrics
  metrics_path: "/metrics"         # metrics 路径
```

### 3. 架构

```
┌─────────────────────────────────────────────────────────────┐
│                     trafficlog.Collector                     │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │   LVSStats   │───>│  gatherStats │───>│  logOutput   │  │
│  │   Adapter    │    │   (共用)      │    │  (原有功能)   │  │
│  └──────────────┘    └──────────────┘    └──────────────┘  │
│                              │                              │
│                              ▼                              │
│                       ┌──────────────┐                       │
│                       │  metrics.Set  │  ← 新增：更新指标值   │
│                       │  (Counter/Gauge)                     │
│                       └──────────────┘                       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                admin.Server (admin_address)                  │
├─────────────────────────────────────────────────────────────┤
│                    GET /metrics                              │
│                 promhttp.Handler()                           │
└─────────────────────────────────────────────────────────────┘
```

## 实施任务

1. 创建 `pkg/metrics` 包 - 定义所有指标和更新方法
2. 创建 `pkg/admin` 包 - HTTP server 暴露 `/metrics`
3. 扩展 `pkg/config/config.go` - 添加 admin 和 metrics 配置字段
4. 修改 `pkg/trafficlog/collector.go` - 收集后更新流量指标
5. 修改 `pkg/healthcheck/manager.go` - 状态变更更新健康指标
6. 修改 `pkg/lvs/reconciler.go` - 错误时递增计数器
7. 修改 `pkg/config/config.go` - 重载成功递增计数器
8. 修改 `pkg/server/server.go` - 初始化 admin server
9. 添加 prometheus 依赖并验证测试
