# ezlb 日志增强与流量可观测性方案

## 背景
- 日志需要统一遵循 `global.log` 配置块（`level`, `home`, `max_size`, `max_backups`, `max_age`, `compress` 等）。
- 系统日志（System）、流量日志（Traffic）、NAT 日志（NAT）应写入日志目录下的独立文件，其中系统日志还需同步到 stdout/stderr 以便容器/系统采集，流量/NAT 日志仅落盘。
- 需要以极低开销获取业务流量可见性：依托 IPVS/SNAT 现有统计信息，按照服务自定义的日志级别（包含 `none`）周期性输出增量流量日志。
- 交付物包含：代码改动、`docs/plans/13.日志增强与流量可观测性方案.plan.md` 方案更新、覆盖新能力的单测与 e2e 测试、README 等文档更新。

## 目标
1. 建立统一的日志配置与 logger 构建方式，保证所有模块字段一致、便于排障。
2. 增加低开销的流量日志能力，支持服务粒度的日志级别及 SNAT 统计。
3. 完成相关文档与测试，确保功能在 macOS/Linux 等环境均可验证。

## 实施计划

### 1. 日志增强
1. **配置结构（`pkg/config/config.go`）**
   - 新增顶层 `LogConfig`，字段：`level`, `home`, `max_size`, `max_backups`, `max_age`, `compress`。
   - 校验日志级别仅允许 `debug/info/warn/error`，service 级别额外允许 `none` 以跳过流量日志。
2. **Logger 初始化（`cmd/ezlb/main.go`）**
   - `buildLoggers` 使用 `zapcore.NewTee` + `lumberjack` 创建三套 logger：
     - **System Logger**：写 stdout/stderr + `${home}/ezlb.log`（若 `home` 非空）。
     - **Traffic Logger**：写 `${home}/traffic.log`，必要时回落到 stdout。
     - **NAT Logger**：写 `${home}/nat.log`，必要时回落到 stdout。
   - CLI 可保留 `--log-level` 对系统级别进行一次性覆盖。
3. **统一字段辅助方法（`pkg/logutil/fields.go`）**
   - 提供 `ServiceFields(serviceCfg)`、`BackendFields(serviceCfg, backendCfg)`，统一 `service`, `listen`, `protocol`, `scheduler`, `backend` 等字段，方便所有模块引用。
4. **模块接入**
   - 核心模块使用 root logger；流量统计组件使用 traffic logger；SNAT 相关操作（含统计）使用 NAT logger。
   - 引入字段辅助方法以保持日志结构一致。
5. **测试**
   - 单测覆盖 `LogConfig` 默认值、非法 level 校验、不同 `home` 配置下的 core 组合。
   - 验证字段辅助方法输出。

### 2. 低开销流量日志
1. **配置扩展（`pkg/config/config.go`）**
   - `ServiceConfig` 新增 `traffic_log_level`，默认继承 `global.log.level`，允许 `debug/info/warn/error/none`。
   - 新增 `global.traffic_logging`，字段包含 `enabled`, `interval`, `include_snat`（默认启用、默认间隔 30s，校验最小值如 5s）。
2. **统计收集组件（`pkg/telemetry/trafficlogger/`）**
   - 依赖 `lvs` stats provider 及可选 `snat` stats provider。
   - 按服务配置决定是否输出及对应的 zap level（`none` 表示跳过）。
   - 维护快照，按间隔计算增量，仅在有变化时写入 traffic logger。
3. **SNAT 统计（`pkg/snat/snat.go`, `pkg/snat/manager_linux.go`）**
   - Manager 接口新增 `Stats()`；Linux 实现解析 `iptables -t nat -vnL EZLB-SNAT`（或 nft），Fake 实现可返回空统计。
   - NAT 相关日志使用 NAT logger。
4. **Server 集成（`pkg/server/server.go`）**
   - 启动时创建 traffic collector goroutine，注入 traffic/nat logger 与配置。
   - `config.OnChange()` 时仅刷新使能、间隔与 service 级别；`shutdown()` 时停止 collector。
5. **测试**
   - 单测使用 fake stats provider 验证增量计算、`none` 级别跳过、SNAT 解析。
   - e2e：启用短间隔 traffic logging，检查 `traffic.log`、`nat.log` 内容与文件生成；禁用或设 `none` 后确认文件不再增长。

### 3. 文档与计划产物

2. 更新 `README` / `README_CN`，新增 `log` 配置块、`traffic_logging`、`traffic_log_level` 的说明及日志文件介绍。
3. 若存在 CHANGELOG/Release Note，记录可观测性增强与新增配置项。

## 验证计划
1. 运行 `go test ./pkg/...`，覆盖配置解析、logger 构建、字段辅助、traffic logger、SNAT 解析等单元测试。
2. 针对 Linux 专属逻辑使用 build tag，确保 macOS/Linux CI 均可通过。
3. 运行新增的 `go test ./tests/e2e -run TestTrafficLogging`，验证守护模式下系统/traffic/nat 日志文件行为及 service 级别控制。
4. 手工冒烟：`sudo ezlb start -c config.yaml`，检查 `${log.home}` 下的 `ezlb.log` / `traffic.log` / `nat.log`，并通过修改配置验证日志级别与流量日志开关的生效情况。
